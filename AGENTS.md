# Second Mind Repo Rules

## Obsidian Linking Rule

When creating or updating notes for the Second Mind Obsidian vault:

1. Do not stop at creating the note file and frontmatter.
2. Every new research or investment note must also be connected from the relevant system page.
3. The connection must be visible in Obsidian through at least one of:
   - a Dataview table that includes `file.link`
   - an explicit wiki link such as `[[Note Name]]`
4. After syncing notes into the live vault, verify that the target system page can expose the new note as a clickable entry rather than only showing metadata fields.

Rationale:

- A file that exists but is not surfaced through `file.link` or wiki links appears "unconnected" to the user.
- In Second Mind, visible navigability is part of the feature, not a cosmetic extra.
