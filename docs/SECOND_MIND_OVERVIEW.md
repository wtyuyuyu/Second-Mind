# SECOND_MIND_OVERVIEW

## What Second Mind Is

Second Mind is the user's local-first personal cognitive operating system.

It is built to coordinate:

- chat
- notes
- papers
- project folders
- decisions
- actions
- reviews
- durable memory

into one visible and evolving system.

Second Mind is not designed as a passive archive.  
It is designed as a working cognitive system that helps the user think, decide, act, and reflect.

## Core Definition

Second Mind = a personal cognitive operating system with:

- local Markdown as the source of truth
- Codex as the compiler
- Obsidian as the browser
- a dashboard as the control console
- action and review as the end goal

## The Core Loop

```text
information intake -> AI filtering -> judgment -> action -> memory -> review -> better future decisions
```

This loop is the center of the entire system.

If a feature does not strengthen this loop, it should not be prioritized in v1.

## Why Second Mind Is Different

Traditional note tools mainly store information.  
RAG systems mainly retrieve information.  
LM Wiki systems mainly maintain structured knowledge.

Second Mind goes further.

It tries to digitize the user's thinking process itself:

- what matters
- what belongs where
- what affects what
- what should be done next
- what should be remembered
- what should be reconsidered later

This makes Second Mind closer to a personal operating system than a conventional wiki.

## Four Systems

Second Mind organizes the user's world into four cooperating systems.

### Research System
Focuses on:

- policy and industry demand
- long-term research directions
- active project progress
- paper and technology trends
- experiment planning
- scientific decision support

### Life System
Focuses on:

- planning and reflection
- personality patterns
- communication habits
- exercise and recovery
- emotional regulation
- relationships
- longer-term life design

### Investment System
Focuses on:

- cash flow
- asset allocation
- relevant company and market signals
- decision support
- risk reminders
- review after decisions

### AI System
Focuses on:

- tools
- local automation
- Codex workflows
- memory vault health
- API and model usage
- the evolution of Second Mind itself

## Cross-System Collaboration

The four systems are not independent silos.

- Research influences Investment through technology, industry, companies, and policy.
- Investment influences Life through risk tolerance, cash-flow pressure, and time horizon.
- Life influences Research through energy, attention, communication quality, and sustainable rhythm.
- AI supports all systems through automation, retrieval, summarization, memory, and execution workflows.

Second Mind should explicitly surface high-value cross-system implications, but should avoid noisy over-linking.

## Why It Is Not a Graph-First Product

Second Mind does not use graph visualization as its central interface.

The reason is simple: graph views are often impressive but hard to read.

Instead, Second Mind uses:

- a tree-like structure for navigation and ownership
- selective cross-links for important relationships
- actions and reviews for operational clarity

In short:

- the tree helps the user see where something belongs
- the links help the user see what else it affects
- the action layer helps the user decide what to do

## Product Direction

The dashboard is important, but it is not the source of truth.

The source of truth remains local Markdown.  
The dashboard is a control console, not the memory substrate itself.

This separation matters:

- Markdown keeps memory portable, auditable, and human-readable
- Codex keeps the system compilable and maintainable
- the dashboard makes the system legible and operational

## V1 Focus

Second Mind v1 does not try to build everything at once.

It focuses on one question:

How can one domain become a strong closed loop from input to review?

The first answer is: the Research System.

## What V1 Will Not Do Yet

To keep the system strong and simple, v1 deliberately excludes:

- automatic mentor scheduling
- graph visualization
- multi-system auto-linked pages
- overly broad source ingestion
- complex autonomous agent orchestration

These may be added later, but they are not required to prove the core value of the system.

## What Success Looks Like

Second Mind succeeds when it helps the user do the following repeatedly:

- identify high-value inputs
- turn them into structured understanding
- produce better next actions
- remember what matters
- review and improve over time

That is the real meaning of a personal cognitive operating system.
