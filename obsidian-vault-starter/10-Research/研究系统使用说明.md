# 研究系统使用说明

## 核心结论

**第一批不需要很多论文，需要的是“少量高信号论文 + 用户自己的判断标记”。**  
Research System 不是论文仓库，而是一个把文献压缩成方向判断和实验动作的系统。

## 第一批应该给什么

最合适的第一批不是 50 篇，而是 **5-8 篇**。  
结构建议如下：

1. **本人已发表论文 1-2 篇**
   - 这是最重要的先验资产
   - 它们不是背景材料，而是已被验证的研究原则

2. **当前研究方向最近邻核心论文 2-3 篇**
   - 和当前主线最接近
   - 最能决定“现在该不该继续做”

3. **反向或对照论文 1-2 篇**
   - 例如不同机制、不同材料、不同 sensing route
   - 用来防止方向判断只在单一叙事里打转

4. **方法 / 工具 / 综述论文 1 篇**
   - 帮用户补研究地图，而不是只补结果

## 直接扔到 Zotero 行不行

**可以。第一步直接扔进 Zotero 就行。**

但只“扔进去”还不够。最小可运行动作应该是：

1. 放进 Zotero
2. 给重要论文加星标
3. 高亮关键段落
4. 每篇至少补一条用户自己的短 note

也就是说：

$$
\text{只导入} < \text{导入 + 标星} < \text{导入 + 标星 + 高亮 + 用户判断}
$$

真正让系统开始工作的，是最后一步。

## 用户自己的想法标记在哪

**最好的位置，是直接标在 Zotero 这篇论文对应的 note 里。**

原因很简单：

- 论文内容和用户判断绑在一起，不会丢
- 后续导入 Obsidian 时，Codex 更容易把“原文证据”和“用户判断”一起抽出来
- 比只在聊天里说一次更 durable

如果一时不想写得很完整，也可以只用下面这套极简标记：

### 推荐标记格式

```text
[一句话结论]
[用户判断]
[为什么重要]
[疑问]
[实验想法]
[归类标签]
```

### 示例

```text
[一句话结论] 这篇文章的关键不是器件本身，而是接触早期编码。
[用户判断] 和深海泰勒流触觉很相关，可迁移的是时序处理，不可直接迁移的是环境介质。
[为什么重要] 能直接影响下一步实验设计。
[疑问] 水下噪声会不会把前期时间窗信息淹没？
[实验想法] 设计一个只看前 50 ms 的水下接触分类实验。
[归类标签] #深海泰勒流 #早期时序处理 #触觉 #可迁移
```

## 如果不想在 Zotero 里写，直接告诉 Codex 行不行

**也可以。**

但两种方式的性质不同：

### 直接告诉 Codex

优点：

- 快
- 适合刚开始、还没形成习惯时使用

缺点：

- 容易散在对话里
- 后续回看不如 Zotero / Obsidian durable

### 标在 Zotero / Obsidian

优点：

- 结构更稳定
- 可回溯
- 适合后续自动分类和长期记忆

缺点：

- 一开始比口头说多一步

所以更实用的规则是：

- **着急时**：先直接告诉 Codex
- **长期沉淀时**：再补回 Zotero note 或 Obsidian 卡片

## Codex 会怎么帮用户分类

Codex 后续会按至少四个维度分类：

### 1. 和当前方向的关系

- 核心相关
- 邻近相关
- 背景支持
- 反向 / 对照

### 2. 证据类型

- 机制论文
- 方法论文
- 实验论文
- 综述 / 地图
- 工具 / 数据 / 平台

### 3. 行动价值

- 高：会直接改变下一步实验或方向判断
- 中：帮助补地图，但不直接决定动作
- 低：可存档，但当前不应占用太多注意力

### 4. 用户立场

- 已认可
- 有保留
- 强怀疑
- 待验证

## 用户第一批最该怎么做

最简起步法只有三步：

1. 把 **5-8 篇**论文放进 Zotero
2. 每篇至少补一条 `[用户判断]`
3. 把最重要的 **3 篇**告诉 Codex

Codex 再负责：

- 帮用户分类
- 生成 `research-paper-source` 卡
- 连到 `research-direction` 页
- 把输入压成下一步实验动作

## Unknown Unknowns

用户可能还没显式意识到的一点是：  
**研究系统最稀缺的不是论文，而是“用户自己的判断标签”。**

因为同样一篇论文：

- 对别人可能只是背景
- 对用户可能是方向转折点

真正决定系统质量的，不是文献数量，而是：

$$
\text{系统质量} \approx \text{高信号输入} + \text{用户判断密度} + \text{可执行动作压缩能力}
$$

## 当前建议

用户现在就可以做这一批：

1. 本人已发表论文 1 篇
2. 深海泰勒流触觉最近邻论文 2 篇
3. 水下触觉 / 柔性传感邻近论文 2 篇
4. 一篇对照或反向路线论文

只要用户把这 **6 篇左右** 放进 Zotero，并给每篇补一句 `[用户判断]`，Research System 就能开始真正工作。
