# 面向大语言模型 agent 的可查询事件化传感器

## 方向定义

这个方向的目标不是“传感器接了 LLM”，而是：

**让传感器从一开始就适配 agent 的工作方式，支持被问询、被调用、补测、自解释和事件化输出。**

## 为什么重要

- 传统传感器默认面对固定算法和连续采样
- agent 系统面对的是多工具协同、主动提问和上下文推理
- 如果这个方向成立，传感器的接口范式本身会改变

## 当前证据

- handoff 已明确：这条线仍偏中长期和概念验证
- 当前最小原型已收敛到：**多模态、事件化、小而硬的触觉探头**

## 关键不确定性

- 还没有最小可行硬件原型
- 评价体系还没有完全成形
- “agent 是否更有效完成任务”如何被量化仍需定义

## 实验或验证路径

1. 定义最小三模态事件探头
2. 规定 agent 可发起的动作，例如轻触、横扫、重复测量
3. 在一个小任务中证明“可查询传感器”优于被动连续读数

## 下一步动作

- [ ] 列出 agent 可以调用的最小动作集合
- [ ] 定义首个最小任务与成功指标

## Unknown Unknowns

这条线真正激进的地方，不是更高精度，而是：

$$
\text{感知器件} \rightarrow \text{可查询工具}
$$

如果这个转变成立，后续很多“怎么设计传感器”的问题都会被重写。
